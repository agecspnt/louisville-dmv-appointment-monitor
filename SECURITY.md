# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Use private reporting channels (GitHub Security Advisories or maintainer contact
defined in repository settings) and include:

- Affected version/commit
- Reproduction steps
- Impact assessment
- Suggested remediation (if available)

We will acknowledge receipt as quickly as possible and work on a fix based on
severity.

## Scope Notes

- This project automates browser checks against third-party services.
- Report vulnerabilities in this repository's code, dependencies, or release artifacts.
- For vulnerabilities in external services, report directly to their owners.
